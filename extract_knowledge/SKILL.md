# extract_knowledge Skill

## 什么时候使用这个 Skill

当有以下内容需要提取知识点时使用：
- PDF/PPT/DOCX 解析后的文本内容
- Markdown 或文本文件内容
- 图片的描述文字

## AI 执行步骤

### 步骤 1: 分析内容结构

仔细阅读提供的文本内容，识别：
- **章节标题和层级**：识别 "第X章"、"X.X" 等标题格式，提取章节名
- **关键概念和定义**：带有 "定义"、"是指"、"称为" 等关键词的句子
- **公式和定理**：数学表达式、物理定律、计算公式
- **例题和案例**：带有 "例"、"案例"、"计算" 等标识的内容
- **图表说明**：表格、图示的描述和说明
- **表格数据**：材料分类表、参数表、对比表等

### 步骤 2: 提取章节信息

**重要**：必须从内容中识别章节归属，不要默认使用 "未分类"。

识别章节的方法：
1. 查找 "第X章"、"Chapter X" 等标题
2. 查找 "X.X" 格式的小节标题
3. 查找文档开头的目录或大纲
4. 根据内容主题推断章节

章节命名规范：
- 使用完整章节名，如 "第1章 绪论"、"第2章 工程材料"
- 如果无法确定章节，使用内容主题作为章节名

### 步骤 3: 提取知识点

对每个识别的知识点，提取以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| title | 知识点标题 | "集合的定义" |
| description | 详细描述（50-200字） | "集合是把具有共同特征的事物..." |
| category | 分类 | 概念/定义/公式/例题/定理/表格 |
| related_concepts | 相关概念 | ["元素", "子集"] |
| chapter | 所属章节 | "第1章 集合" |

### 步骤 4: 特殊内容提取

#### 4.1 表格提取

当遇到表格时，转换为结构化格式：


识别知识点之间的关系：
- 前置知识关系
- 相关概念关联
- 章节归属

## 知识点分类标准

### 概念 (概念)
- 基本概念和术语
- 学科基础概念
- 方法论概念

### 定义 (定义)
- 明确的定义陈述
- 规范化的术语解释
- 标准定义

### 公式 (公式)
- 数学公式
- 计算公式
- 物理定律表达式

### 例题 (例题)
- 典型例题
- 应用案例
- 解题示范

### 定理 (定理)
- 数学定理
- 物理定律
- 重要结论

### 表格 (表格)
- 材料分类表
- 参数对照表
- 特性对比表

## 提取原则

1. **准确性**: 确保提取的知识点准确反映原文内容
2. **完整性**: 不要遗漏重要的知识点，包括表格和计算过程
3. **简洁性**: 描述要简洁明了，避免冗余
4. **关联性**: 正确识别知识点之间的关联关系
5. **层次性**: 保持知识点的层次结构
6. **章节归属**: 必须为每个知识点指定正确的章节

## 注意事项

- 如果内容过长，分批处理，每次处理约 8000 字符
- 保持知识点的独立性，每个知识点应该可以单独理解
- 避免重复提取相同的知识点
- 对于公式，使用 LaTeX 格式表示
- **必须为每个知识点指定正确的章节归属**
- 表格内容要完整提取，不要遗漏
- 计算题要提取完整的已知条件和求解步骤
